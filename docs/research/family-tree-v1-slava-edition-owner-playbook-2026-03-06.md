## 1. Для кого этот документ

1. Для владельца дерева.
2. Для человека, который будет реально пользоваться системой после запуска.
3. Цель документа - дать короткий и понятный operational guide без технической терминологии.

## 2. Базовая модель доступа

1. У дерева есть три основных типа доступа:
- `Владелец` - полный контроль над деревом,
- `Администратор` - может помогать с редактированием и загрузкой файлов,
- `Участник` - доступ по аккаунту без права редактирования.
2. Отдельно есть `семейная ссылка`:
- она не выдает роль,
- она подходит только для просмотра,
- ее можно в любой момент отозвать.

## 3. Что использовать в какой ситуации

### 3.1 Когда отправлять семейную ссылку

1. Когда родственнику нужно просто посмотреть дерево.
2. Когда не хочется просить человека регистрироваться.
3. Когда нужен быстрый read-only доступ.

### 3.2 Когда приглашать участника

1. Когда человек должен войти в систему под своим доступом.
2. Когда человеку нужен постоянный доступ.
3. Когда нужно видеть, кто именно из участников помогает с деревом.

### 3.3 Когда делать человека администратором

1. Когда родственник помогает редактировать людей, связи и файлы.
2. Когда родственник должен сам загружать семейные фотографии, видео или документы.
3. Когда человеку нужно не только смотреть, но и участвовать в ведении архива.

## 4. Типовые действия владельца

### 4.1 Добавить нового человека

1. Откройте `Конструктор`.
2. Выберите `Человек`.
3. Заполните имя и основные поля.
4. Сохраните запись.
5. При необходимости свяжите нового человека через кнопки добавления родителя, ребенка или партнера.

### 4.2 Пригласить помощника

1. Откройте `Участники`.
2. В блоке приглашений выберите роль:
- `Участник`, если нужен только доступ по аккаунту,
- `Администратор`, если человек будет помогать вести дерево.
3. Создайте приглашение.
4. Отправьте полученную ссылку человеку.

### 4.3 Отправить дерево родственнику без регистрации

1. Откройте `Участники`.
2. В блоке `Семейные ссылки` создайте новую ссылку.
3. Скопируйте адрес сразу после создания.
4. Отправьте ссылку родственнику.
5. Если доступ больше не нужен, отзовите ссылку на том же экране.

### 4.4 Загрузить файл для конкретного человека

1. Откройте `Конструктор`.
2. Выберите человека.
3. Используйте нужную вкладку:
- `Фото` для фотографий,
- `Видео` для видео,
- `Инфо` для документов и общих данных о человеке.
4. Выберите локальные файлы.
5. На экране проверки:
- удалите лишнее,
- при необходимости добавьте еще файлы,
- подтвердите сохранение кнопкой `Сохранить`.
6. Дождитесь завершения загрузки.

### 4.5 Загрузить файл в общий семейный архив

1. Откройте раздел `Медиа` в навигации дерева.
2. Выберите режим `Фото` или `Видео`.
3. При необходимости выберите существующий альбом или создайте новый.
4. Выберите файлы и подтвердите пакет после экрана проверки.
5. Используйте этот раздел для материалов, которые не нужно сразу привязывать к одному человеку.

## 5. Рекомендуемая практика видимости

1. `Публично` использовать только для материалов, которые можно спокойно показывать получателям семейной ссылки.
2. `Только участникам` использовать для личных и чувствительных файлов.
3. Если есть сомнение, по умолчанию лучше выбирать `Только участникам`.

## 6. Что делать при проблемах

### 6.1 Если родственник не открывает дерево по ссылке

1. Проверьте, не отозвана ли ссылка.
2. Проверьте, не истек ли срок действия.
3. Если ссылка старая или сомнительная, создайте новую и отправьте ее заново.

### 6.2 Если помощник не может редактировать дерево

1. Проверьте его роль на экране `Участники`.
2. Если у него роль `Участник`, а нужен edit-доступ, повысьте до `Администратора`.
3. Если доступ больше не нужен, отзовите его и при необходимости создайте новое приглашение.

### 6.3 Если файл не загружается

1. Убедитесь, что файл не слишком большой для V1.
2. Попробуйте загрузить файл еще раз.
3. Если ошибка повторяется, зафиксируйте:
- кто загружал,
- какой файл,
- где это произошло: `Фото`, `Видео`, `Инфо` или tree-level `Медиа`,
- текст ошибки на экране.

## 7. Что видно в журнале

1. Создание и отзыв семейных ссылок.
2. Приглашения и изменения ролей.
3. Создание, изменение и удаление людей.
4. Создание и удаление файлов в person media и tree-level archive.
5. Для семейных ссылок в V1 достаточно видеть `когда создана/отозвана` и `last accessed`, без шумного логирования каждого открытия.

## 8. Простое правило эксплуатации

1. Для просмотра без регистрации используйте `семейную ссылку`.
2. Для постоянного доступа используйте `приглашение`.
3. Для реальной помощи с деревом используйте роль `Администратора`.
4. Для person media используйте вкладки `Фото`, `Видео` и `Инфо`.
5. Для общесемейных материалов используйте tree-level раздел `Медиа`.
6. Если доступ больше не нужен, лучше отозвать его сразу, чем оставлять открытым.
