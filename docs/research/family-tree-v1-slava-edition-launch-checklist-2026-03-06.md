# Family Tree V1.0 "Slava Edition" Launch and UAT Checklist (2026-03-06)

## 1. Цель

1. Этот документ нужен перед private production launch.
2. Его задача - собрать минимальный checklist запуска без лишнего process overhead.

## 2. Технический pre-launch

### 2.1 Конфигурация

1. Проверены production env values.
2. Проверен `NEXT_PUBLIC_SITE_URL`.
3. Проверен storage bucket.
4. Проверен server role key.

### 2.2 База и миграции

1. Все нужные миграции применены.
2. Есть backup перед launch.
3. Таблицы для:
- `tree_memberships`,
- `tree_invites`,
- `tree_share_links`,
- `media_assets`,
- `person_media`
  находятся в ожидаемом состоянии.

### 2.3 Smoke на код

1. `npm run typecheck`
2. `npm test`
3. Ключевые smoke/e2e готовы к прогону.

## 3. Product UAT

### 3.1 Владелец из Европы

1. Входит в систему.
2. Открывает builder.
3. Добавляет человека.
4. Добавляет связь.
5. Загружает файл.
6. Создает приглашение.
7. Создает семейную ссылку.
8. Отзывает семейную ссылку.

### 3.2 Помощник из РФ

1. Принимает приглашение.
2. Входит в дерево.
3. Открывает builder.
4. Может редактировать дерево.
5. Может загрузить файл.
6. Не может управлять тем, что не должен по роли.

### 3.3 Read-only родственник из РФ

1. Открывает семейную ссылку.
2. Видит дерево без логина.
3. Видит разрешенные файлы.
4. Не попадает в builder как editor.
5. После revoke ссылки больше не открывает дерево.

## 4. Access and privacy check

1. `viewer` не может редактировать.
2. `share-link` пользователь не может редактировать.
3. `admin` может помогать с деревом и файлами.
4. `owner` видит журнал и управляет доступами.
5. Файлы `members` не светятся неподходящим получателям.

## 5. Media check

1. Фото открываются.
2. Видео открываются.
3. Документы открываются.
4. Удаление файла работает.
5. После удаления не остается явно битого состояния в UI.

## 6. Audit and operational check

1. В журнале видно:
- создание приглашения,
- принятие приглашения,
- изменение роли,
- отзыв доступа,
- создание семейной ссылки,
- отзыв семейной ссылки,
- создание/удаление файла.
2. `last_accessed_at` по семейным ссылкам обновляется.
3. Ошибки на ключевых действиях отображаются пользователю.

## 7. Network and geography check

1. Viewer и builder работают из Европы.
2. Viewer, invite flow и media reading работают из РФ.
3. Если automated smoke не проходит из-за сети, это фиксируется как operational note, а не скрывается.

## 8. Launch decision rule

Запуск разрешен, если:
1. Владелец может сам вести дерево.
2. Помощник может реально помогать.
3. Родственник может читать дерево по ссылке.
4. Файлы работают по private delivery.
5. Есть backup/restore дисциплина.

Запуск откладывается, если:
1. Не подтвержден доступ из одной из географий.
2. Семейные ссылки ведут себя нестабильно.
3. Upload/read media ломается в реальном сценарии.
4. Нет понятного recovery path.
